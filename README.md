# Lead Lifecycle ADF Repository

## Overview

This repository contains Azure Data Factory (ADF) artifacts for the Lead Lifecycle incremental data pipeline.

The pipeline performs incremental extraction from Azure Database for PostgreSQL and loads data into Snowflake.

## Architecture

Azure Database for PostgreSQL (Flexible Server)
→ Azure Data Factory (AutoResolveIntegrationRuntime)
→ Snowflake (LEADS table)

## Pipeline Components

Pipeline:
- PL_IncrementalLoad_PostgresToSnowflake

Trigger:
- TR_Incremental_Leads_30Min (runs every 30 minutes)

Linked Services:
- Azure PostgreSQL
- Snowflake
- Azure Blob Storage (staging)

Datasets:
- DS_Postgres_Leads
- DS_Snowflake_Leads
- DS_Snowflake_Watermark

## Incremental Load Logic

1. Lookup current watermark from PIPELINE_WATERMARK
2. Extract records where UPDATED_DATE_UTC > watermark
3. Copy data to Snowflake LEADS
4. Update watermark after successful load

This ensures:
- No duplicates
- Efficient incremental processing
- Idempotent execution

## Git Integration

This repository is connected to Azure Data Factory in Git mode.

- main branch → Development artifacts (datasets, pipelines, triggers)
- adf_publish branch → ARM template for deployment

Artifacts under /adf are automatically generated by ADF.

## Deployment

To deploy the factory:

1. Publish from ADF Studio
2. Use ARM template from adf_publish branch

Note: This repository contains only ADF artifacts.  
Business logic transformation into lifecycle events is implemented in a separate Python project.
